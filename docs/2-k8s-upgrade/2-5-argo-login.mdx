# Argo Workflow 재배포 + 로그인하기

## helm chart 수정

이전에는 따로 수정 없이 helm chart를 배포했었음  
이번에는 간단하게 helm chart를 수정

1. `fullnameOverride` 원하는 이름으로 변경 `myargo`

2. Master node에 배포하기 위해 `nodeSelector` 옵션을 변경  
   controller.nodeSelector 와 server.nodeSelector에 다음 라벨을 추가
   `node-role.kubernetes.io/master: "true"`

   라벨 옵션은 다음 명령어로 확인

   ```
   k get nodes --show-labels
   ```

   라벨 설정은 여기서 다루지 않겠습니다.

3. 접근을 위한 LoadBalancer 설정

```yaml title="values.yaml" {5,7,11,28}
(...)
server:
  (...)
  # -- Service type for server pods
  serviceType: LoadBalancer
  # -- Service port for server
  servicePort: 80
  # -- Service node port
  serviceNodePort: # 32746
  # -- Service port name
  servicePortName: "http" # http

  serviceAccount:
    # -- Create a service account for the server
    create: true
    # -- Service account name
    name: ""
    # -- Labels applied to created service account
    labels: {}
    # -- Annotations applied to created service account
    annotations: {}

  # -- Annotations to be applied to the UI Service
  serviceAnnotations: {}
  # -- Optional labels to add to the UI Service
  serviceLabels: {}
  # -- Static IP address to assign to loadBalancer service type `LoadBalancer`
  loadBalancerIP: "192.168.0.200"
(...)
```

## 로그인을 위한 Account 설정하기

SSO 방식도 있지만, 여기서는 token 로그인 방식[^1]을 적용해 보겠습니다.

token을 기본 계정에서 추출하지 않고 새로 계정을 만들자.  
(기본 계정에서 token 추출이 가능할 수도 있지만, 알아서 할 것)

yaml 3개 작성
`huadmin`은 이름에 맞게 변경
그리고 여기서는 Argo에서 기본 생성하는 ClusterRole을 이용하였으니
Role을 따로 만들고 싶으면 yaml 하나 더 작성해서 binding 해주면 됨

```yaml title="rb-admin.yaml"
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: argooperatorrolebinding
  namespace: {{ .Release.Namespace | quote }}
subjects:
- kind: ServiceAccount
  name: huadmin
roleRef:
  kind: ClusterRole
  name: {{ template "argo-workflows.fullname" . }}-admin
  apiGroup: rbac.authorization.k8s.io
```

```yaml title="sa-admin.yaml"
apiVersion: v1
kind: ServiceAccount
metadata:
  name: huadmin
  namespace: { { .Release.Namespace | quote } }
secrets:
  - name: huadmin-sa
```

```yaml title="secret-admin-token.yaml"
apiVersion: v1
kind: Secret
metadata:
  name: huadmin-secret
  namespace: { { .Release.Namespace | quote } }
  annotations:
    kubernetes.io/service-account.name: huadmin
type: kubernetes.io/service-account-token
```

## 배포 후 접속하기

설정이 끝났으니 다시 Argo Workflow를 배포해 봅시다.  
Chart가 위치한 폴더로 이동해 다음 명령어를 실행합니다.

```
helm install my-argowf ./argo-workflows -n argo-wf --create-namespace
```

토큰 확인
Linux는 그대로 하면 되고, Windows의 경우 master VM Shell에 들어가서 확인했음

```
ARGO_TOKEN="Bearer $(sudo k3s kubectl get secret huadmin-secret -n argo-wf -o=jsonpath='{.data.token}' | base64 --decode)"
echo $ARGO_TOKEN
```

값을 저장해 둔다

![kubectl check](./img/2-5-kubectl.png)

정상적으로 고정 IP가 할당되었습니다.  
브라우저로 해당 IP에 접속하면 로그인 화면이 나옵니다.

![login page](./img/2-5-argo-1.png)

여기서 아까 저장해 두었던 토큰을 이용해 로그인을 진행합니다.

![login success](./img/2-5-argo-2.png)

정상적으로 로그인되었습니다!
